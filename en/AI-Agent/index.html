About the Project

This project implements a local AI agent using Ollama and vector databases for intelligent document retrieval and question answering. The agent runs entirely on your local machine, leveraging ChromaDB for efficient semantic search over CSV documents and personal data.

Built with Ollama for local LLM inference, the system combines natural language processing with vector embeddings to provide context-aware responses based on stored documents, including personal information and datasets like restaurant reviews.

Key Features

  • Local LLM with Ollama: Runs language models locally without external API dependencies
  • Vector Database Integration: Uses ChromaDB for efficient semantic search and document retrieval
  • Document Processing: Processes and indexes multiple CSV files including personal information and domain-specific datasets
  • Semantic Search: Performs intelligent semantic queries over stored documents
  • Context-Aware Responses: Retrieves relevant information from vector store to provide informed answers
  • Privacy-Focused: All data and processing stays on your local machine

How It Works

  1. Documents and CSV files are loaded into the system
  2. Text is split into chunks and converted to vector embeddings
  3. Embeddings are stored in ChromaDB for efficient retrieval
  4. User queries are converted to vectors and matched against the database
  5. Relevant documents are retrieved based on semantic similarity
  6. Ollama LLM generates responses using the retrieved context

Dataset Components

  • Personal Information: Sahil Thakur's biographical and professional data
  • Restaurant Reviews: Dataset for sentiment analysis and information retrieval
  • Custom Data: Additional CSV datasets for specific use cases
